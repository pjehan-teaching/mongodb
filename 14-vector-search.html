Vector Search

Embeddings

Les embeddings sont des représentations numériques des objets (comme les mots, les phrases, ou les images) dans un espace vectoriel continu.

Les embeddings sont utilisés pour capturer les relations sémantiques entre les objets. Par exemple, dans le cas des mots, les mots similaires auront des embeddings similaires.

Représentation vectorielle

A = [3.00, 2.00]
B = [−1.00, 3.00]

Chaque vecteur est une liste de nombres représentant un point dans un espace multidimensionnel.

En recherche vectorielle, on stocke des embeddings (vecteurs à des centaines de dimensions) et on les compare pour trouver les éléments les plus sémantiquement proches.

⟵ Faites glisser les extrémités des vecteurs

Calcul de similarité

Le calcul de similarité permet de mesurer la proximité entre deux vecteurs. Il existe plusieurs méthodes :

  • Euclidean Distance : distance en ligne droite entre deux points
  • Manhattan Distance : distance par chemin orthogonal
  • Dot Product : produit scalaire (direction + magnitude)
  • Cosine Similarity : angle entre les vecteurs (direction uniquement)

Distance Euclidienne

d = √((bx − ax)² + (by − ay)²)
A = [−2.00, −1.00]
B = [2.00, 3.00]
√((4.00)² + (4.00)²)
Résultat : 5.66

Distance minimale entre deux points (le chemin le plus court en ligne droite).

⟵ Faites glisser les points A et B

Distance de Manhattan

d = |bx − ax| + |by − ay|
A = [−2.00, −1.00]
B = [2.00, 3.00]
|4.00| + |4.00|
Résultat : 8.00

Somme des déplacements sur chaque axe (comme naviguer dans un quadrillage de rues).

⟵ Faites glisser les points A et B

Produit Scalaire (Dot Product)

a·b = ax × bx + ay × by
A = [3.00, 1.00]
B = [1.00, 3.00]
3.00 × 1.00 + 1.00 × 3.00
Résultat : 6.00

> 0 : même direction · = 0 : perpendiculaires · < 0 : directions opposées

⟵ Faites glisser les extrémités des vecteurs

Similarité Cosinus

cos(θ) = (a·b) / (|a| × |b|)
A = [3.00, 1.00]
B = [1.00, 3.00]
θ = 36.87°
Résultat : 0.80

Valeur entre −1 (opposés) et 1 (identiques). Indépendante de la magnitude (seule la direction compte).

utils.ts

import { pipeline, FeatureExtractionPipeline } from '@huggingface/transformers';

const MODEL = 'Xenova/paraphrase-multilingual-MiniLM-L12-v2';
let extractorPromise: Promise<FeatureExtractionPipeline> | null = null;

export async function getExtractor(): Promise<FeatureExtractionPipeline> {
  if (!extractorPromise) {
    console.log("Chargement du modèle d'embedding...");
    extractorPromise = pipeline('feature-extraction', MODEL) as Promise<FeatureExtractionPipeline>;
    const ext = await extractorPromise;
    console.log("Modèle prêt.");
    return ext;
  }
  return extractorPromise;
}

export async function generateEmbedding(text: string): Promise<number[]> {
  const ext = await getExtractor();
  // Pour paraphrase-multilingual-MiniLM-L12-v2, la combinaison pooling: 'mean' + normalize: true est celle recommandée dans la documentation du modèle sur HuggingFace.
  const output = await ext(text, {
    pooling: 'mean',    // moyenne des vecteurs de chaque token → un seul vecteur par phrase
    normalize: true,    // normalisation L2 : ramène la norme du vecteur à 1, ce qui rend la similarité cosinus équivalente au produit scalaire
  });
  return Array.from(output.data as Float32Array);
}

export function cosineSimilarity(a: number[], b: number[]): number {
  const dot   = a.reduce((sum, ai, i) => sum + ai * b[i], 0);
  const normA = Math.sqrt(a.reduce((sum, ai) => sum + ai ** 2, 0));
  const normB = Math.sqrt(b.reduce((sum, bi) => sum + bi ** 2, 0));
  return dot / (normA * normB);
}

01_embeddings.ts

import { generateEmbedding, cosineSimilarity } from './utils.js';

const phrases = [
  "J'aime jouer au football.",
  "Le football est mon sport favori.",
  "J'adore manger des pâtes.",
  "La cuisine italienne est ma préférée.",
];

const embeddings = await Promise.all(phrases.map(generateEmbedding));
console.log(`Dimensions : ${embeddings[0].length}\n`);

for (let i = 0; i < phrases.length; i++) {
  for (let j = i + 1; j < phrases.length; j++) {
    const sim = cosineSimilarity(embeddings[i], embeddings[j]);
    console.log(`[${sim.toFixed(4)}] "${phrases[i]}" ↔ "${phrases[j]}"`);
  }
}

export interface Article {
  titre: string;
  contenu: string;
  categorie: string;
  auteur: string;
  embedding?: number[];
}

export async function getCollection<T extends object>(name: string) {
  const client = new MongoClient(process.env.MONGODB_URI!);
  await client.connect();
  const collection = client.db('vector_search').collection<T>(name);
  return { client, collection };
}

02_insert.ts — Connexion & insertion

import { Article, generateEmbedding, getCollection } from './utils.js';

const { client, collection } = await getCollection<Article>('articles');

try {
  await collection.deleteMany({});           // repart de zéro

  for (const article of articles) {
    article.embedding = await generateEmbedding(
      `${article.titre} ${article.contenu}`  // titre + contenu → embedding
    );
    console.log(`✓ ${article.titre}`);
  }

  await collection.insertMany(articles);
  console.log(`\n${articles.length} articles insérés.`);
} finally {
  await client.close();
}

Étape 3 — Index de recherche vectorielle

L'index vectoriel indique à MongoDB comment comparer les embeddings. Il doit être créé une seule fois avant de pouvoir effectuer des recherches.

03_index.ts

import { Article, getCollection } from './utils.js';

const { client, collection } = await getCollection<Article>('articles');

try {
  await collection.createSearchIndex({
    name: 'vector_index',
    type: 'vectorSearch',
    definition: {
      fields: [
        {
          type: 'vector',
          path: 'embedding',
          numDimensions: 384,    // doit correspondre aux dimensions du modèle
          similarity: 'cosine',  // cohérent avec normalize: true dans utils.ts
        },
        {
            type: "filter",
            path: "categorie",       // Permet de filtrer par catégorie lors des recherches
        },
      ],
    },
  });
  console.log('Index "vector_index" créé avec succès.');
} finally {
  await client.close();
}

Étape 4 — Recherche vectorielle

L'étape finale consiste à interroger MongoDB avec un vecteur de requête via l'opérateur d'agrégation $vectorSearch.

L'outil CLI accepte deux arguments : la requête en langage naturel et, optionnellement, une catégorie pour affiner les résultats. 

npm run 04 -- "recette de cuisine italienne"
npm run 04 -- "sport et bien-être" sport

04_search.ts

import { Article, generateEmbedding, getCollection } from './utils.js';

const query     = process.argv[2];
const categorie = process.argv[3]; // optionnel : filtre par catégorie

if (!query) {
  console.error('Usage: npm run 04 -- "<requête>" [<catégorie>]');
  process.exit(1);
}

const { client, collection } = await getCollection<Article>('articles');

try {
  const queryEmbedding = await generateEmbedding(query);

  const results = await collection.aggregate<Article & { score: number }>([
    {
      $vectorSearch: {
        index: 'vector_index',
        path: 'embedding',
        queryVector: queryEmbedding,
        numCandidates: 50,
        limit: 5,
        ...(categorie ? { filter: { categorie } } : {}), // filtre optionnel
      },
    },
    {
      $project: {
        _id: 0, titre: 1, categorie: 1, auteur: 1,
        score: { $meta: 'vectorSearchScore' }, // score de similarité cosinus
      },
    },
  ]).toArray();

  const label = categorie ? ` (catégorie : ${categorie})` : '';
  console.log(`\nRésultats pour "${query}"${label}:\n`);
  results.forEach((r, i) => {
    console.log(`${i + 1}. [${r.categorie}] ${r.titre} — ${r.auteur}`);
    console.log(`   Score : ${r.score.toFixed(4)}\n`);
  });
} finally {
  await client.close();
}
